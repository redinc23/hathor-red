# Incident Response Runbook

## Severity Levels

- SEV1: Full outage or data loss
- SEV2: Major feature impaired
- SEV3: Degraded performance
- SEV4: Minor issue

## Triage Checklist

- Confirm incident and impact scope
- Identify affected services
- Check recent deploys and feature flags
- Establish communication channel

## Response Steps

1. Assign incident commander.
2. Roll back or disable feature flags if needed.
3. Mitigate user impact quickly.
4. Document timeline and decisions.

## Communication

- Update status page at least every 30 minutes
- Notify stakeholders with ETA and mitigation

## Postmortem

- Create RCA within 48 hours
- Track corrective actions
