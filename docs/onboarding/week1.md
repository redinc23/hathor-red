# Week 1 Onboarding Guide

## Goals

- Understand the architecture and data flow
- Ship a small, user-visible improvement
- Learn the release and incident workflows

## Suggested Plan

Day 1-2
- Read `ARCHITECTURE.md` and `API.md`
- Walk through critical user flows
- Review open issues and tech debt

Day 3-4
- Pair on a small bug fix
- Add or update a test
- Read existing runbooks in `docs/runbooks/`

Day 5
- Deliver your first PR
- Ask for feedback on code conventions
